# PRD-005 — Administração de Banners e Categorias

| Campo | Valor |
|--------|--------|
| **Projeto** | ERP Comercial Grifres |
| **Versão** | 1.0.0 |
| **Status** | Aprovado |
| **Prioridade** | Crítica |

---

# 1. Objetivo

Implementar o painel administrativo para gerenciamento de **Banners e Categorias**, permitindo que todo o conteúdo exibido na Home e navegação da loja seja controlado dinamicamente pelo sistema.

Este PRD conecta o frontend público (Home, Banners, Categorias) ao backend administrativo.

---

# 2. Objetivos

- Criar CRUD completo de Banners.
- Criar CRUD completo de Categorias.
- Persistir dados no banco.
- Substituir dados mockados por dados reais.
- Permitir controle total via painel administrativo.
- Garantir que mudanças reflitam imediatamente na loja.

---

# 3. Escopo

Este PRD contempla:

## Banners
- Criar banner
- Editar banner
- Listar banners
- Ativar/desativar banner
- Definir ordem de exibição
- Definir tipo de banner (Hero, Institucional, Promocional)

## Categorias
- Criar categoria
- Editar categoria
- Listar categorias
- Ativar/desativar categoria
- Definir ordem
- Upload/URL de imagem

---

Não contempla:

- Produtos
- Variantes
- Pedidos
- Clientes
- Estoque
- Financeiro

# 7. Integração com Frontend Público

### Home

- Substituir banners mockados por API `/banners`
- Substituir categorias mockadas por API `/categories`

### Reatividade

- Atualização imediata após salvamento.
- Cache controlado via TanStack Query.


- Modal de edição

---

# 10. Regras de Negócio

- Banner inativo não aparece na loja.
- Categoria inativa não aparece na loja.
- Ordem sempre respeitada no frontend.
- Slug de categoria deve ser único.
- Não permitir categorias órfãs inválidas (se usar hierarquia futura).

# 16. Observações
