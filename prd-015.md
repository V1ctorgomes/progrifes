# PRD-015 — Entradas de Estoque

| Campo | Valor |
|--------|--------|
| **Projeto** | ERP Comercial Grifres |
| **Versão** | 1.0.0 |
| **Status** | Aprovado |
| **Prioridade** | Crítica |

---

# 1. Objetivo

Implementar o módulo de Entradas de Estoque do ERP Comercial Grifres.

Este módulo será responsável por registrar toda movimentação que aumente o saldo de estoque das variantes, mantendo rastreabilidade completa e histórico de todas as entradas.

Nenhum saldo poderá ser alterado diretamente no módulo de estoque. Toda alteração deverá ocorrer através de uma movimentação registrada.

---

# 2. Objetivos

- Registrar entradas de estoque.
- Atualizar automaticamente o saldo disponível.
- Manter histórico de movimentações.
- Identificar a origem da entrada.
- Garantir rastreabilidade completa.

---

# 3. Escopo

Este PRD contempla:

- Cadastro de entradas.
- Atualização automática do estoque.
- Histórico de movimentações.
- Pesquisa.
- Filtros.
- Impressão do comprovante de entrada.

Não contempla:

- Compras de fornecedores.
- Inventário.
- Ajustes de estoque.
- Transferências.

Esses recursos serão implementados em PRDs específicos.

---

# 16. Regras de Negócio

- Não permitir quantidade menor ou igual a zero.
- Toda entrada deverá gerar uma movimentação.
- Não permitir exclusão física.
- Apenas usuários autorizados poderão registrar entradas.
- O saldo do estoque nunca poderá ser alterado manualmente.

---

# 20. Critérios de Aceitação

O PRD será considerado concluído quando:

- Cadastro de entradas funcionando.
- Estoque atualizado automaticamente.
- Histórico registrado.
- Pesquisa funcionando.
- Filtros funcionando.
- Impressão disponível.
- Auditoria funcionando.

---

# 21. Fora do Escopo

Não implementar:

- Ordem de compra.
- Recebimento de mercadorias.
- Inventário.
- Ajustes negativos.
- Transferências entre estoques.

Esses recursos serão implementados em PRDs posteriores.

---

# 22. Observações

Este módulo inaugura o conceito de movimentações oficiais do estoque.

A partir dele, nenhuma alteração de saldo deverá ocorrer diretamente na tabela de estoque.

Toda alteração deverá possuir uma movimentação registrada, garantindo rastreabilidade completa, auditoria e confiabilidade das informações do ERP Comercial Grifres.
