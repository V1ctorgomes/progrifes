# PRD-014 — Controle de Estoque

| Campo | Valor |
|--------|--------|
| **Projeto** | ERP Comercial Grifres |
| **Versão** | 1.0.0 |
| **Status** | Aprovado |
| **Prioridade** | Crítica |

---

# 1. Objetivo

Implementar o módulo central de Controle de Estoque do ERP Comercial Grifres.

Este módulo será responsável por controlar o saldo de todas as variantes cadastradas, registrar reservas, controlar disponibilidade para venda e fornecer informações precisas sobre o estoque em tempo real.

Todo o controle de estoque será realizado por **variante**, nunca pelo produto principal.

---

# 2. Objetivos

- Centralizar o estoque do ERP.
- Controlar saldo das variantes.
- Controlar reservas.
- Disponibilizar estoque para a loja.
- Identificar produtos com estoque baixo.
- Preparar a estrutura para movimentações, inventários e compras.

---

# 3. Escopo

Este PRD contempla:

- Controle de saldo.
- Reserva de estoque.
- Liberação de reserva.

- Estoque mínimo.

Não contempla:

- Entradas.
- Saídas.
- Inventário.
- Compras.
- Ajustes manuais.

Esses recursos serão implementados nos próximos PRDs.

---

# 4. Conceitos

Cada variante possuirá quatro quantidades distintas.

```text
Estoque Total

↓

Estoque Reservado

↓

Estoque Disponível

↓

Estoque Mínimo
```

Onde:

```text
Disponível = Total - Reservado
```

---

# 5. Estrutura do Estoque

Cada variante deverá possuir:

```text
Variante

SKU

Quantidade Total

Quantidade Reservada

Quantidade Disponível

Estoque Mínimo

Status
```

---

# 6. Status do Estoque

Criar os seguintes status:

```text
Em Estoque

Estoque Baixo

Sem Estoque
```

Regras:

**Em Estoque**

```text
Disponível > Estoque Mínimo
```

**Estoque Baixo**

```text
Disponível <= Estoque Mínimo
```

**Sem Estoque**

```text
Disponível = 0
```

O status será calculado automaticamente.

---

# 7. Reserva de Estoque

Quando um pedido for criado:

```text
Pedido

↓

Reserva Estoque

↓

Disponível diminui

↓

Reservado aumenta
```

Enquanto o pedido estiver ativo, o estoque permanecerá reservado.

---

# 8. Liberação da Reserva

Caso o pedido seja cancelado:

```text
Reservado diminui

↓

Disponível aumenta
```

Sem alterar o estoque total.

---

# 9. Baixa Definitiva

Quando o pedido for entregue:

```text
Total diminui

↓

Reservado diminui
```

A movimentação será registrada automaticamente.

# 19. Regras de Negócio

- Nunca permitir estoque negativo.
- Nunca permitir reservas superiores ao estoque disponível.
- Toda alteração deverá ser registrada no histórico de movimentações.
- Apenas variantes ativas poderão possuir estoque disponível.
- O cálculo do estoque disponível será automático.

# 25. Observações

Este módulo será o núcleo do controle operacional do ERP Comercial Grifres.

A partir dele, todas as áreas do sistema utilizarão uma única fonte de verdade para consulta de disponibilidade dos produtos.

Pedidos, compras, inventários, financeiro, relatórios e dashboards dependerão diretamente deste módulo, tornando-o um dos componentes mais importantes de toda a arquitetura do sistema.
