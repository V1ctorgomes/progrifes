# PRD-011 — Gestão de Pedidos

| Campo | Valor |
|--------|--------|
| **Projeto** | ERP Comercial Grifres |
| **Versão** | 1.0.0 |
| **Status** | Aprovado |
| **Prioridade** | Crítica |

---

# 1. Objetivo

Implementar o módulo completo de Gestão de Pedidos do ERP Comercial Grifres.

Este módulo será o centro operacional da loja, permitindo que os colaboradores acompanhem todo o ciclo de vida de um pedido, desde sua criação até a entrega ao cliente.

Toda alteração de status será realizada exclusivamente pelo painel administrativo.

---

# 2. Objetivos

- Centralizar todos os pedidos.
- Gerenciar o fluxo operacional.
- Atualizar status.
- Consultar histórico.
- Preparar integração com entregadores.
- Preparar integração com o financeiro.
- Preparar integração com controle de estoque.

---

# 3. Escopo

Este PRD contempla:

- Listagem de pedidos.
- Visualização completa.
- Alteração de status.
- Timeline.
- Histórico.
- Pesquisa.
- Filtros.
- Impressão de pedido.
- Cancelamento.

Não contempla:

- Financeiro.
- Emissão de nota fiscal.
- Integração com entregadores.
- Integração com ERP externo.

---

# 4. Fluxo Operacional

```text
Pedido Criado

↓

Aguardando Confirmação

↓

Confirmado

↓

Separando Pedido

↓

Pronto para Entrega

↓

Saiu para Entrega

↓

Entregue
```

Fluxo alternativo:

```text
Pedido Criado

↓

Cancelado
```

---

# 5. Status

Criar os seguintes status:

```text
AGUARDANDO_CONFIRMACAO

CONFIRMADO

SEPARANDO

PRONTO_PARA_ENTREGA

SAIU_PARA_ENTREGA

ENTREGUE

CANCELADO
```

Cada status deverá possuir:

- nome;
- cor;
- descrição;
- ordem.

---

# 6. Listagem

Criar página:

```text
/admin/pedidos
```

Exibir:

- Número do pedido
- Cliente
- Telefone
- Data
- Quantidade de itens
- Valor total
- Forma de pagamento
- Status
- Ações

---

# 7. Pesquisa

Permitir pesquisa por:

- Número do pedido
- Nome do cliente
- Telefone

Busca em tempo real.

---

# 8. Filtros

Implementar:

- Status
- Data
- Forma de pagamento
- Bairro
- Entregador (estrutura preparada)
- Valor

---

# 9. Ordenação

Permitir ordenar por:

- Mais recentes
- Mais antigos
- Maior valor
- Menor valor
- Status

```text
Dropdown

ou

```

- data;
- hora;
- usuário responsável.

# 13. Impressão

Criar versão para impressão.

Conteúdo:

- dados do cliente;
- endereço;
- itens;
- observações;
- pagamento.

Formato otimizado para impressoras térmicas e A4.


O PRD será considerado concluído quando:

- Listagem funcionando.

---

# 23. Fora do Escopo

Não implementar:

- WhatsApp API.
- Emissão de NF-e.
- Integração com financeiro.
- Integração com estoque.
- Integração com entregadores.
- Notificações automáticas.

Esses recursos serão desenvolvidos nos próximos PRDs.

---

# 24. Observações

Este módulo será o principal centro operacional do ERP Comercial Grifres.

Todas as áreas do sistema — estoque, financeiro, entregas, relatórios e CRM — utilizarão os pedidos como base para suas operações.

O histórico de alterações deverá ser imutável, permitindo rastrear todas as ações realizadas pelos usuários e garantindo maior controle operacional e auditoria.
