# PRD-001 — Home

| Campo | Valor |
|--------|--------|
| **Projeto** | ERP Comercial Grifres |
| **Versão** | 1.0.0 |
| **Status** | Aprovado |
| **Prioridade** | Crítica |

---

# 1. Objetivo

Desenvolver a Home da loja virtual do ERP Comercial Grifres.

A Home será a principal porta de entrada do sistema para os clientes, apresentando a identidade da loja, produtos em destaque, categorias, promoções e informações institucionais.

Neste PRD serão utilizadas imagens públicas (URLs) apenas para desenvolvimento.

Todo o layout deverá ser desenvolvido do zero, sem utilização de bibliotecas de componentes.

---

# 2. Objetivos

- Desenvolver uma Home moderna.
- Construir uma interface totalmente responsiva.
- Criar uma estrutura preparada para futuras integrações.
- Permitir gerenciamento posterior dos conteúdos através do painel administrativo.

---

# 3. Escopo

Este PRD contempla apenas o frontend da Home.

Não contempla:

- Painel administrativo
- Cadastro de banners
- Cadastro de produtos
- Cadastro de categorias
- Carrinho
- Checkout
- WhatsApp
- Autenticação

Os dados poderão ser mockados durante o desenvolvimento.

---

# 4. Estrutura da Página

A Home deverá possuir a seguinte estrutura.

```text
Navbar

↓

Hero Banner

↓

Categorias em Destaque

↓

Produtos em Destaque

↓

Coleções

↓

Benefícios

↓

Banner Institucional

↓

Produtos Recentes

↓

Chamada para WhatsApp

↓

Rodapé
```

---

# 5. Navbar

A Navbar deverá conter:

### Logo

Posicionada à esquerda.

Inicialmente utilizar um texto como placeholder.

---

### Menu

Itens:

- Início
- Produtos
- Categorias
- Sobre
- Contato

---

### Pesquisa

Campo de busca centralizado.

Neste PRD não possuirá funcionalidade.

---

### Ícones

Direita da Navbar.

- Pesquisa (Mobile)
- Favoritos (placeholder)
- Carrinho

O contador do carrinho poderá utilizar valor fictício.

---

### Mobile

A Navbar deverá possuir menu hamburguer.

---

# 6. Hero Banner

Primeira seção da página.

Deverá conter:

- imagem grande;
- título;
- subtítulo;
- botão principal;
- botão secundário.

Utilizar imagens públicas via URL.

---

# 7. Categorias em Destaque

Exibir categorias em formato de cards.

Cada card deverá possuir:

- imagem;
- nome;
- quantidade de produtos (mock).

Inicialmente:

- Masculino
- Feminino
- Infantil
- Promoções

---

# 8. Produtos em Destaque

Grid responsivo.

Cada produto deverá conter:

- imagem;
- nome;
- preço;
- preço promocional (quando existir);
- selo de promoção;
- botão de visualizar;
- botão adicionar ao carrinho.

Utilizar dados mockados.

---

# 9. Coleções

Seção destinada para futuras campanhas.

Exemplo:

```text
Coleção Inverno

↓

Imagem

↓

Descrição

↓

Botão
```

Inicialmente utilizar conteúdo fictício.

---

# 10. Benefícios

Apresentar os principais diferenciais da loja.

Exemplo:

- Entrega rápida
- Atendimento via WhatsApp
- Produtos de qualidade
- Pagamento facilitado

Cada benefício deverá possuir:

- ícone;
- título;
- descrição.

---

# 11. Banner Institucional

Seção horizontal destacando a marca.

Exemplo:

- imagem;
- título;
- descrição;
- botão.

---

# 12. Produtos Recentes

Mesmo padrão dos produtos em destaque.

Utilizar produtos diferentes dos anteriores.

---

# 13. Chamada para WhatsApp

Seção incentivando o cliente a entrar em contato.

Conteúdo:

Título

Texto

Botão:

"Falar no WhatsApp"

Neste PRD o botão não precisará possuir integração.

---

# 14. Rodapé

O Footer deverá conter:

## Informações

- Logo
- Descrição da empresa

---

## Links

- Produtos
- Categorias
- Sobre
- Contato

---

## Contato

- WhatsApp
- Endereço
- Horário

Utilizar informações fictícias.

---

## Redes Sociais

Ícones:

- Instagram
- Facebook
- WhatsApp

---

## Direitos Autorais

Linha inferior.

---

# 15. Componentes

Todos deverão ser desenvolvidos manualmente.

Componentes previstos:

- Navbar
- Mobile Menu
- Hero
- Section Title
- Category Card
- Product Card
- Collection Card
- Benefit Card
- Banner
- WhatsApp CTA
- Footer
- Button
- Input
- Badge
- Price
- Container

---

# 16. Responsividade

Obrigatório funcionamento em:

- Desktop
- Notebook
- Tablet
- Smartphone

Mobile First.

---

# 17. Acessibilidade

Implementar:

- HTML semântico
- Labels
- Navegação por teclado
- Contraste adequado
- Textos alternativos nas imagens
- Estados de foco

---

# 18. Performance

- Lazy Loading das imagens
- Utilização do componente Image do Next.js
- Otimização de fontes
- Componentização
- Evitar renderizações desnecessárias

---

# 19. Dados Mockados

Criar dados temporários para:

- Produtos
- Categorias
- Coleções
- Benefícios
- Banner principal

Todas as imagens poderão utilizar URLs públicas.

---

# 20. Critérios de Aceitação

A Home será considerada concluída quando possuir:

- Navbar responsiva.
- Hero Banner.
- Categorias em destaque.
- Produtos em destaque.
- Coleções.
- Benefícios.
- Banner institucional.
- Produtos recentes.
- Chamada para WhatsApp.
- Rodapé completo.
- Layout totalmente responsivo.
- Componentes reutilizáveis.
- Código organizado seguindo os padrões definidos no PRD-000.

---

# 21. Fora do Escopo

Não implementar:

- Backend.
- Banco de dados.
- API.
- CRUD.
- Autenticação.
- Painel administrativo.
- Carrinho funcional.
- Favoritos.
- Busca funcional.
- Integração com WhatsApp.
- SEO.

Esses recursos serão implementados nos próximos PRDs.

---

# 22. Observações

A Home deverá representar o padrão visual que será utilizado em todo o ERP Comercial Grifres.

Todo o desenvolvimento deve priorizar reutilização de componentes, organização do código e preparação para futuras integrações com o backend, sem depender de dados reais nesta etapa.
