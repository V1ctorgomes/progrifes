# PRD-017 — Inventário de Estoque

| Campo | Valor |
|--------|--------|
| **Projeto** | ERP Comercial Grifres |
| **Versão** | 1.0.0 |
| **Status** | Aprovado |
| **Prioridade** | Crítica |

---

# 1. Objetivo

Implementar o módulo de Inventário de Estoque do ERP Comercial Grifres.

Este módulo permitirá realizar conferências físicas do estoque, comparar as quantidades reais com as registradas no sistema e gerar ajustes auditáveis quando houver divergências.

Nenhuma alteração manual de saldo será permitida fora do processo de inventário.

---

# 2. Objetivos

- Realizar inventários completos ou parciais.
- Comparar estoque físico e sistema.
- Registrar divergências.
- Gerar ajustes automáticos.
- Manter histórico completo.
- Garantir rastreabilidade.

---

# 3. Escopo

Este PRD contempla:

- Inventário completo.
- Inventário parcial.
- Contagem física.
- Recontagem.
- Ajustes automáticos.
- Histórico.
- Relatório de divergências.

Não contempla:

- Leitura por código de barras.
- Coletor de dados.
- Multiestoque.
- Inventário cíclico automatizado.

---

# 4. Tipos de Inventário

O sistema deverá suportar:

```text
Inventário Geral

Inventário Parcial

Inventário por Categoria

Inventário por Produto

Inventário por Variante
```


```text

```

---

# 7. Contagem Física

Para cada variante será exibido:

```text
Produto

Variante

SKU

Quantidade no Sistema

Campo para Quantidade Física
```

O operador informará apenas a quantidade encontrada.

---

# 8. Divergências

Após finalizar a contagem:

O sistema deverá calcular automaticamente:

```text
Quantidade Sistema

↓

Quantidade Física

↓

Diferença

↓

Ajuste Necessário
```

---

# 9. Ajustes

Ao confirmar o inventário:

Caso exista diferença positiva:

```text
Gerar Entrada de Estoque
```

Caso exista diferença negativa:

```text
Gerar Saída de Estoque
```

Todos os ajustes deverão gerar movimentações automáticas.

---

# 10. Recontagem

Antes da confirmação final:

Permitir:

```text
Recontar Item

ou

Recontar Todo Inventário
```

Enquanto houver recontagem, nenhum ajuste será aplicado.

# 19. Regras de Negócio

- Apenas um inventário poderá estar em andamento por vez.
- Não permitir finalizar sem concluir a contagem.
- Todo ajuste deverá gerar movimentação automática.
- Inventários finalizados não poderão ser alterados.
- O histórico será permanente.

# 23. Critérios de Aceitação

O PRD será considerado concluído quando:

- Criação de inventários funcionando.
- Contagem física funcionando.
- Comparação automática funcionando.
- Divergências identificadas.

- Relatório funcionando.

---

# 24. Fora do Escopo

Não implementar:

- Leitor de código de barras.
- RFID.
- Multiestoque.
- Inventário cíclico automático.
- Integração com coletores de dados.

Esses recursos poderão ser adicionados futuramente.

---

# 25. Observações

O módulo de Inventário de Estoque será o único mecanismo autorizado para corrigir divergências entre o estoque físico e o estoque registrado no sistema.

Todos os ajustes serão convertidos automaticamente em movimentações oficiais de entrada ou saída, preservando a integridade do histórico e garantindo auditoria completa.

Com este PRD, o módulo de estoque do ERP Comercial Grifres passa a estar funcionalmente completo, oferecendo controle de saldo, reservas, movimentações, entradas, saídas e inventários, formando uma base sólida para os módulos de compras, financeiro e relatórios.
