# PRD-027 — Configuração da Entrega Própria

| Campo | Valor |
|--------|--------|
| **Projeto** | ERP Comercial Grifres |
| **Versão** | 1.0.0 |
| **Status** | Aprovado |
| **Prioridade** | Alta |

---

# 1. Objetivo

Implementar o módulo de Configuração da Entrega Própria, responsável por centralizar todas as configurações relacionadas às entregas realizadas pela própria loja.

Este módulo será utilizado pelos demais módulos do ERP, como Checkout, Pedidos e Gestão de Entregas.

---

# 2. Objetivos

- Configurar a operação de entrega da loja.
- Definir horários de funcionamento para entregas.
- Configurar prazo médio de entrega.
- Definir valor mínimo para entrega (opcional).
- Configurar mensagens exibidas ao cliente.
- Preparar integração com o cadastro de bairros.

---

# 3. Escopo

Este PRD contempla:

- Configurações gerais da entrega.
- Horários de funcionamento.
- Dias de atendimento.
- Tempo médio de entrega.
- Valor mínimo para pedidos (opcional).
- Mensagens personalizadas.
- Ativação ou desativação das entregas.

Não contempla:

- Cadastro de bairros.
- Taxas de entrega.
- Controle de entregadores.
- Rastreamento.

---

# 4. Configurações Gerais

Criar uma tela de configurações contendo:

```text
Entrega Própria Ativa

Sim / Não
```

Quando desativada:

- Não será possível realizar pedidos com entrega.

---

# 5. Horário de Funcionamento

Permitir configurar:

```text
Segunda-feira

Horário Inicial

Horário Final

--------------------------------

Terça-feira

...

--------------------------------

Domingo
```

Cada dia poderá ser:

- Ativo
- Inativo

---

# 6. Prazo Médio

Permitir configurar:

```text
Tempo Médio

30 minutos

45 minutos

60 minutos

90 minutos

120 minutos
```

Também permitir informar qualquer valor manualmente.

---

# 7. Pedido Mínimo

Permitir definir:

```text
Pedido mínimo para entrega

R$ 0,00
```

Caso o pedido seja inferior ao mínimo:

```text
Pedido não poderá ser finalizado.
```

---

# 8. Mensagens

Cadastrar mensagens que serão exibidas no site.

Exemplo:

```text
Entrega realizada pela própria loja.

Prazo médio: 45 minutos.

Consulte bairros atendidos.
```

Outra mensagem:

```text
No momento estamos fora do horário de entrega.
```

---

# 9. Administração

Criar página:

```text
/administrativo/configuracoes/entrega
```

---

## Funcionalidades

- Editar configurações.
- Salvar.
- Restaurar padrões.
- Ativar/desativar entregas.

---

# 14. Regras de Negócio

- Apenas administradores poderão alterar as configurações.
- O sistema deverá validar se a loja está dentro do horário de funcionamento.
- Caso a entrega esteja desativada, o checkout deverá ocultar a opção "Entrega".
- O pedido mínimo deverá ser validado antes da finalização da compra.

---

# 15. Segurança

Implementar:

- JWT.
- Guards.
- Controle de permissões.
- Auditoria das alterações.

---

# 16. Performance

- Cache das configurações.
- Cache dos horários.
- Carregamento único das configurações por sessão.

---

# 17. Responsividade

Obrigatório:

- Desktop.
- Notebook.
- Tablet.
- Smartphone.

---

# 18. Critérios de Aceitação

O PRD será considerado concluído quando:

- Configurações funcionando.
- Horários funcionando.
- Prazo médio funcionando.
- Pedido mínimo funcionando.
- Mensagens funcionando.
- Integração com Checkout funcionando.
- Integração com Pedidos preparada.

---

# 19. Fora do Escopo

Não implementar:

- Cadastro de bairros.
- Taxas de entrega.
- Controle de entregadores.
- Rastreamento.
- Cálculo automático de frete.

Esses recursos serão implementados nos próximos PRDs.

---

# 20. Observações

Este módulo será a base da logística do ERP Comercial Grifres.

Todas as regras relacionadas às entregas próprias serão centralizadas neste módulo, permitindo que o sistema opere de forma padronizada e que futuras expansões, como múltiplas lojas ou diferentes modalidades de entrega, possam ser implementadas sem necessidade de refatoração da arquitetura existente.
