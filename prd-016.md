# PRD-016 — Saídas e Movimentações de Estoque

| Campo | Valor |
|--------|--------|
| **Projeto** | ERP Comercial Grifres |
| **Versão** | 1.0.0 |
| **Status** | Aprovado |
| **Prioridade** | Crítica |

---

# 1. Objetivo

Implementar o módulo de Saídas e Movimentações de Estoque do ERP Comercial Grifres.

Este módulo será responsável por registrar todas as movimentações que reduzam o estoque físico, mantendo rastreabilidade completa e garantindo que nenhuma baixa ocorra sem registro.

Além das saídas manuais, este módulo também centralizará todas as movimentações geradas automaticamente pelo sistema.

---

# 2. Objetivos

- Registrar saídas manuais.
- Registrar perdas e avarias.
- Registrar consumo interno.
- Atualizar automaticamente o estoque.
- Centralizar o histórico de movimentações.
- Garantir auditoria completa.

---

# 3. Escopo

Este PRD contempla:

- Saídas manuais.
- Perdas.
- Avarias.
- Consumo interno.
- Doações.
- Ajustes negativos.
- Histórico de movimentações.

Não contempla:

- Inventário.
- Transferência entre estoques.
- Compras.
- Recebimento de mercadorias.

---

# 4. Tipos de Saída

O sistema deverá suportar:

```text
Venda

Perda

Avaria

Consumo Interno

Doação

Ajuste Negativo

Outros
```

O tipo **Venda** será utilizado automaticamente pelos pedidos entregues.

# 14. Movimentações Automáticas

O sistema deverá registrar automaticamente movimentações quando ocorrer:

- Pedido entregue.
- Pedido cancelado (liberação de reserva).
- Entrada de estoque.
- Inventário.
- Ajustes futuros.

Nenhuma movimentação automática poderá ser removida manualmente.

# 18. Regras de Negócio

- Não permitir estoque negativo.
- Não permitir saída maior que o estoque disponível.
- Toda saída deverá gerar movimentação.
- Não permitir exclusão física.
- Toda movimentação deverá ser auditável.

- Inventário.
- Transferências.
- Multiestoque.
- Compras.
- Recebimento de mercadorias.

Esses recursos serão implementados em PRDs posteriores.

---

# 24. Observações

Após este PRD, toda alteração de estoque no ERP Comercial Grifres passará obrigatoriamente pela Central de Movimentações.

Ela se tornará a fonte oficial de auditoria do estoque, permitindo identificar quando, por quem, por qual motivo e em qual operação cada unidade foi movimentada.

Esse modelo elimina alterações diretas de saldo, aumenta a confiabilidade dos dados e prepara o sistema para futuras integrações com compras, inventários, financeiro e relatórios.
