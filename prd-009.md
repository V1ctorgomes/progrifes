# PRD-009 — Carrinho de Compras

| Campo | Valor |
|--------|--------|
| **Projeto** | ERP Comercial Grifres |
| **Versão** | 1.0.0 |
| **Status** | Aprovado |
| **Prioridade** | Crítica |

---

# 1. Objetivo

Implementar o sistema completo de Carrinho de Compras do ERP Comercial Grifres.

O carrinho será responsável por armazenar temporariamente os produtos selecionados pelo cliente, permitindo alterações antes da finalização do pedido via WhatsApp.

Toda a lógica deverá utilizar as **variantes dos produtos**, garantindo que estoque, preço e informações sejam sempre consistentes.

---

# 2. Objetivos

- Adicionar produtos ao carrinho.
- Atualizar quantidades.
- Remover produtos.
- Persistir o carrinho.
- Calcular subtotal.
- Calcular taxa de entrega futuramente.
- Preparar integração com o Checkout via WhatsApp.

---

# 3. Escopo

Este PRD contempla:

- Carrinho completo.
- Persistência do carrinho.
- Atualização de quantidades.
- Remoção de produtos.
- Resumo do pedido.
- Validação de estoque.

Não contempla:

- Finalização do pedido.
- Geração do pedido.
- Pagamentos.
- Entrega.

Esses recursos serão implementados nos próximos PRDs.

# 6. Funcionalidades

Implementar:

- Adicionar produto.
- Alterar quantidade.
- Remover item.
- Esvaziar carrinho.
- Atualização automática do subtotal.
- Atualização automática do total.

---

# 7. Persistência

O carrinho deverá permanecer salvo caso o usuário atualize a página.

Inicialmente utilizar:

```text
Local Storage
```

A arquitetura deverá permitir futura sincronização com usuários autenticados.

---

# 8. Regras de Negócio

- Não permitir adicionar variantes inativas.
- Não permitir quantidade maior que o estoque disponível.
- Caso o estoque seja alterado, validar novamente antes do checkout.
- Produtos iguais (mesma variante) devem ser agrupados.

# 16. Validações

Ao adicionar um item:

- Variante obrigatória.
- Quantidade mínima de 1.
- Quantidade máxima igual ao estoque.

Ao atualizar:

- Não permitir valores negativos.
- Remover automaticamente caso quantidade seja zero.

---

Implementar:

- Context API.
- Memoização dos cálculos.
- Atualização otimizada.
- Persistência eficiente no Local Storage.

---

# 18. Responsividade

---

# 19. Critérios de Aceitação

O PRD será considerado concluído quando:

- Adicionar produto funcionar.
- Atualizar quantidade funcionar.
- Remover item funcionar.
- Carrinho persistir após atualizar a página.

---

# 20. Fora do Escopo

Não implementar:

- Pagamento.
- Geração do pedido.
- Integração com WhatsApp.
- Cálculo de frete.
- Cupons.
- Cashback.
- Pontos.
- Login do cliente.

Esses recursos serão implementados em PRDs futuros.

---

# 21. Observações

Este módulo representa a transição entre a navegação da loja e o processo de compra.

Todo o carrinho deverá utilizar **variantes dos produtos** como unidade de controle, garantindo consistência com estoque, pedidos e movimentações futuras.

Sua arquitetura deverá permitir evolução para sincronização com usuários autenticados, recuperação de carrinhos abandonados e integração com múltiplos canais de venda.
