# 1. Objetivo

Este PRD define toda a fundação técnica do ERP Comercial Grifres.

Nenhuma regra de negócio será implementada neste PRD. Seu único objetivo é estabelecer a arquitetura, tecnologias, padrões de desenvolvimento, segurança, estrutura de pastas e convenções que deverão ser seguidas durante todo o ciclo de vida do projeto.

Todos os próximos PRDs deverão respeitar integralmente este documento.

# 3. Filosofia do Projeto

Todo o sistema deverá ser desenvolvido utilizando componentes próprios.

**Não será utilizada nenhuma biblioteca de componentes de interface.**

Todo componente visual deverá ser desenvolvido manualmente utilizando apenas:

- HTML
- CSS (Tailwind)
- React
- TypeScript

Isso garante:

- identidade visual própria;
- maior flexibilidade;
- ausência de dependências visuais externas;
- facilidade de manutenção.

# 7. Padrões de Código

Obrigatório seguir:

# 8. Design System

Todo o Design System será proprietário.

Não utilizar:

- bibliotecas de componentes;
- templates prontos;
- kits de interface.

Todo componente deverá ser criado pela equipe.

Exemplos:

- Button
- Input
- Select
- Checkbox
- Modal
- Drawer
- Dialog
- Toast
- Tooltip
- Accordion
- Tabs
- Badge
- Card
- Dropdown
- Pagination
- Table
- Breadcrumb
- Sidebar
- Navbar
- Data Grid

Todos serão implementados internamente conforme a necessidade do projeto.

- padrões de código;
- estrutura de pastas;
- convenções de desenvolvimento;
