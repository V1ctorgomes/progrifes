# PRD-007 — Sistema de Variantes (Cor, Tamanho, SKU e Estoque)

| Campo | Valor |
|--------|--------|
| **Projeto** | ERP Comercial Grifres |
| **Versão** | 1.0.0 |
| **Status** | Aprovado |
| **Prioridade** | Crítica |

---

# 1. Objetivo

Implementar o sistema completo de variantes de produtos do ERP Comercial Grifres.

Cada produto poderá possuir uma ou mais variantes, permitindo controlar estoque, preço, SKU e imagens individualmente para cada combinação.

O estoque do ERP será baseado nas variantes, e não no produto principal.

---

# 2. Objetivos

- Permitir múltiplas variantes por produto.
- Controlar estoque por variante.
- Gerar SKU individual.
- Permitir preço por variante.
- Permitir imagens por variante.
- Preparar o sistema para pedidos e movimentação de estoque.

---

# 3. Escopo

Este PRD contempla:

- Cadastro de variantes.
- Gestão de atributos.
- Controle de estoque.
- SKU.
- Código de barras (opcional).
- Imagens da variante.
- Atualização da página do produto.

Não contempla:

- Entradas de estoque.
- Saídas de estoque.
- Inventário.
- Compras.
- Pedidos.

Esses módulos serão implementados posteriormente.

---

# 4. Conceito

Produto:

```text
Camiseta Oversized
```

Variantes:

```text
Preto | P

Preto | M

Preto | G

Branco | P

Branco | M

Branco | G
```

Cada combinação representa uma variante independente.

# 9. SKU

Cada variante possuirá SKU único.

Exemplo:

```text

CAM-001-PT-M

CAM-001-BR-G
```

O administrador poderá editar manualmente.

---

# 25. Observações

Este PRD estabelece a estrutura de estoque do ERP Comercial Grifres.

A partir deste ponto, toda movimentação de estoque, pedidos, compras, inventários e relatórios deverá considerar a **variante** como unidade de controle, e não o produto principal.

Essa abordagem garante maior flexibilidade para produtos com diferentes cores, tamanhos ou outras características, mantendo consistência entre o ERP e a loja virtual.
