# PRD-030 — Acompanhamento das Entregas

| Campo | Valor |
|--------|--------|
| **Projeto** | ERP Comercial Grifres |
| **Versão** | 1.0.0 |
| **Status** | Aprovado |
| **Prioridade** | Alta |

---

# 1. Objetivo

Implementar o módulo de Acompanhamento das Entregas do ERP Comercial Grifres.

Este módulo será responsável por controlar todo o fluxo operacional das entregas realizadas pela própria loja, desde a separação do pedido até sua conclusão, permitindo que a equipe acompanhe em tempo real o status de cada entrega.

O objetivo é fornecer uma visão clara da operação logística diária, facilitando a distribuição de pedidos entre os entregadores e o acompanhamento da produtividade.

---

# 2. Objetivos

- Centralizar todas as entregas da loja.
- Controlar o status operacional das entregas.
- Vincular pedidos aos entregadores.
- Registrar horários de cada etapa.
- Disponibilizar painel operacional.
- Gerar indicadores logísticos.

---

# 3. Escopo

Este PRD contempla:

- Controle de status.
- Atribuição de entregador.
- Timeline da entrega.
- Painel operacional.
- Histórico.
- Pesquisa.
- Filtros.

Não contempla:

- Rastreamento em tempo real.
- GPS.
- Aplicativo para entregadores.
- Notificações automáticas.

---

# 4. Fluxo da Entrega

Todo pedido seguirá obrigatoriamente o fluxo abaixo:

```text
Pedido Recebido

↓

Em Separação

↓

Pronto para Entrega

↓

Saiu para Entrega

↓

Entregue
```

Fluxos alternativos:

```text
Cancelado
```

```text
Não Entregue
```

---

# 5. Status

Cada entrega poderá possuir um dos seguintes status:

```text
Pedido Recebido

Em Separação

Pronto para Entrega

Saiu para Entrega

Entregue

Não Entregue

Cancelado
```

```text

Cliente

Telefone

Bairro

Taxa de Entrega

Entregador

Status

Horário do Pedido

Previsão de Entrega
```

---

# 7. Alteração de Status

Permitir alterar o status da entrega.

Ao alterar para:

```text
Saiu para Entrega
```

O sistema deverá:

- registrar a data e hora;
- atualizar automaticamente o status do entregador para **Em Entrega**.

Ao alterar para:

```text
Entregue
```

O sistema deverá:

- registrar a data e hora;
- atualizar automaticamente o status do entregador para **Disponível**;
- finalizar a entrega.

# 16. Regras de Negócio

- Apenas pedidos com modalidade **Entrega** poderão gerar registros neste módulo.
- Um pedido poderá possuir apenas uma entrega.
- Apenas entregadores ativos poderão ser vinculados.
- Ao concluir a entrega, o entregador retornará automaticamente para o status **Disponível**.
- Todas as alterações deverão ser registradas no histórico.
- Não será permitida exclusão física de entregas.

---

# 21. Fora do Escopo

Não implementar:

- Rastreamento GPS.
- Compartilhamento de localização.
- Aplicativo do entregador.
- Notificações automáticas via WhatsApp.
- Assinatura eletrônica da entrega.

Esses recursos poderão ser adicionados futuramente.

---

# 22. Observações

Este módulo será responsável pelo gerenciamento operacional das entregas da Grifres.

A proposta é oferecer um fluxo simples, eficiente e totalmente integrado ao ERP, permitindo que a equipe acompanhe cada etapa da entrega, distribua pedidos aos entregadores e monitore o desempenho da operação em tempo real.

A arquitetura foi planejada para permitir futuras evoluções, como aplicativo do entregador, rastreamento por GPS, confirmação digital da entrega e notificações automáticas ao cliente, sem necessidade de alterações estruturais.
