# PRD-002 — Sistema de Banners

| Campo | Valor |
|--------|--------|
| **Projeto** | ERP Comercial Grifres |
| **Versão** | 1.0.0 |
| **Status** | Aprovado |
| **Prioridade** | Alta |

---

# 1. Objetivo

Desenvolver o sistema completo de banners do ERP Comercial Grifres.

O sistema deverá permitir que todos os banners exibidos na loja sejam gerenciados posteriormente pelo painel administrativo, sem necessidade de alterações no código.

Neste PRD, os dados poderão ser mockados utilizando imagens públicas por URL.

---

# 2. Objetivos

- Criar um sistema flexível de banners.
- Permitir reutilização em qualquer página.
- Preparar a estrutura para integração futura com o backend.
- Desenvolver componentes reutilizáveis.

---

# 3. Escopo

Este PRD contempla:

- Estrutura dos banners.
- Componentes.
- Exibição na Home.
- Estrutura preparada para integração.

Não contempla:

- Painel administrativo.
- Upload de imagens.
- Banco de dados.
- Backend.
- API.

---

# 4. Tipos de Banner

O sistema deverá suportar os seguintes formatos.

## Hero Banner

Banner principal da Home.

Características:

- imagem desktop;
- imagem mobile;
- título;
- subtítulo;
- botão principal;
- botão secundário;
- link.

---

## Banner Horizontal

Utilizado entre seções.

Características:

- imagem;
- título;
- descrição;
- botão.

---

## Banner Promocional

Banner para campanhas.

Características:

- imagem;
- título;
- chamada;
- botão.

---

## Banner Institucional

Destinado à apresentação da empresa.

Características:

- imagem;
- título;
- descrição.

---

# 5. Estrutura dos Dados

Cada banner deverá possuir:

```text
id

nome

tipo

titulo

subtitulo

descricao

imagemDesktop

imagemMobile

textoBotaoPrimario

textoBotaoSecundario

linkPrimario

linkSecundario

ativo

ordem
```

Inicialmente esses dados serão mockados.

---

# 6. Hero Slider

O Hero deverá suportar múltiplos banners.

Funcionalidades:

- navegação por setas;
- indicadores inferiores;
- autoplay;
- pausa ao passar o mouse;
- suporte ao toque em dispositivos móveis;
- animação suave entre slides.

---

# 7. Responsividade

Desktop:

Utilizar imagem horizontal.

Mobile:

Utilizar imagem específica para dispositivos móveis.

Caso não exista imagem mobile, utilizar a versão desktop.

---

# 8. Componentes

Criar os seguintes componentes.

```text
HeroSlider

HeroSlide

BannerHorizontal

BannerPromocional

BannerInstitucional

BannerButton

BannerIndicators

BannerNavigation
```

Todos deverão ser reutilizáveis.

---

# 9. Estrutura de Pastas

```text
components/

banner/

HeroSlider

HeroSlide

BannerCard

BannerNavigation

BannerIndicators
```

---

# 10. Animações

Implementar animações suaves para:

- troca de slides;
- entrada dos textos;
- botões;
- indicadores.

As animações deverão ser discretas e focadas na experiência do usuário.

---

# 11. Performance

Implementar:

- Lazy Loading das imagens.
- Pré-carregamento do primeiro banner.
- Utilização do componente Image do Next.js.
- Carregamento otimizado das imagens.

---

# 12. Dados Mockados

Criar no mínimo:

## Hero

- 3 banners.

---

## Banner Promocional

- 2 banners.

---

## Banner Institucional

- 1 banner.

Todas as imagens poderão utilizar URLs públicas.

---

# 13. Responsividade

Suporte obrigatório para:

- Desktop
- Notebook
- Tablet
- Smartphone

---

# 14. Preparação para Backend

Todo o sistema deverá ser desenvolvido pensando em futura integração.

Os componentes não poderão depender de dados fixos.

Os dados deverão ser recebidos por propriedades.

Exemplo:

```ts
<HeroSlider banners={banners} />
```

---

# 15. Critérios de Aceitação

O PRD será considerado concluído quando:

- Hero Slider funcionando.
- Navegação entre banners.
- Autoplay.
- Responsividade completa.
- Banner horizontal implementado.
- Banner promocional implementado.
- Banner institucional implementado.
- Componentes reutilizáveis.
- Código organizado.
- Preparado para futura integração com API.

---

# 16. Fora do Escopo

Não implementar:

- CRUD.
- Upload.
- Banco de dados.
- API.
- Painel administrativo.
- Armazenamento em nuvem.
- Permissões.

Esses recursos serão implementados em PRDs futuros.

---

# 17. Observações

Todo o sistema de banners deverá servir como base para qualquer área do ERP Comercial Grifres que necessite exibir conteúdo promocional ou institucional.

A implementação deve priorizar reutilização, baixo acoplamento e facilidade de manutenção, garantindo que a futura integração com o painel administrativo ocorra sem necessidade de refatoração dos componentes.
