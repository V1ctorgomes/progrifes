# PRD-031 — Dashboard Administrativo

| Campo | Valor |
|--------|--------|
| **Projeto** | ERP Comercial Grifres |
| **Versão** | 1.0.0 |
| **Status** | Aprovado |
| **Prioridade** | Muito Alta |

---

# 1. Objetivo

Implementar o Dashboard Administrativo do ERP Comercial Grifres.

Este será o painel inicial do sistema, reunindo em uma única tela todas as informações mais importantes da operação da empresa, permitindo que gestores acompanhem indicadores de vendas, pedidos, estoque, clientes, financeiro e entregas em tempo real.

O dashboard deverá servir como centro de monitoramento do ERP.

---

# 2. Objetivos

- Centralizar os principais indicadores.
- Exibir informações em tempo real.
- Facilitar a tomada de decisão.
- Disponibilizar atalhos para os principais módulos.
- Permitir filtros por período.
- Preparar estrutura para novos indicadores.

---

# 3. Escopo

Este PRD contempla:

- Cards de indicadores.
- Gráficos.
- Listas resumidas.
- Atividades recentes.
- Atalhos rápidos.
- Filtros por período.

Não contempla:

- Relatórios detalhados.
- Exportações.
- BI avançado.
- Dashboards personalizados por usuário.

---

# 4. Página

Criar:

```text
/administrativo/dashboard
```

Esta será a página inicial após o login.

# 6. Cards Principais

Exibir:

```text
Pedidos Hoje

Vendas Hoje

Faturamento do Dia

Clientes Cadastrados

Produtos Cadastrados

Produtos Sem Estoque

Entregas Pendentes

Contas a Receber

Contas a Pagar
```

Cada card deverá possuir:

- título;
- valor;
- ícone;
- comparação com período anterior (quando disponível);
- link para o módulo correspondente.

---

# 18. Banco de Dados

Este módulo **não possuirá tabelas próprias**.

Consumirá informações de:

```text
orders

customers

products

inventory_movements

financial_transactions

deliveries
```

---

# 19. Regras de Negócio

- Todos os dados devem respeitar o período selecionado.
- Os indicadores devem ser atualizados automaticamente ao recarregar a página.
- O dashboard deve respeitar as permissões do usuário.
- Usuários sem acesso ao financeiro não visualizarão dados financeiros.
- Usuários sem acesso ao estoque não visualizarão indicadores de estoque.

---

# 20. Segurança

Implementar:

- JWT.
- Guards.
- Controle de permissões por widget.
- Auditoria de acessos.

---

# 21. Performance

- Cache dos indicadores.
- Cache dos gráficos.
- Consultas otimizadas.
- Índices para consultas frequentes.
- Atualização assíncrona dos widgets.
- Lazy loading dos gráficos.

---

# 22. Responsividade

Obrigatório:

- Desktop.
- Notebook.
- Tablet.
- Smartphone.

Os cards deverão reorganizar-se automaticamente conforme o tamanho da tela.

---

# 23. Critérios de Aceitação

O PRD será considerado concluído quando:

- Dashboard funcionando.
- Cards carregando corretamente.
- Gráficos funcionando.
- Pedidos recentes funcionando.
- Resumo financeiro funcionando.
- Resumo de estoque funcionando.
- Resumo de entregas funcionando.
- Atividades recentes funcionando.
- Filtros funcionando.
- Permissões respeitadas.

---

# 24. Fora do Escopo

Não implementar:

- Dashboard personalizável.
- Widgets arrastáveis.
- Exportação em PDF.
- Exportação em Excel.
- BI avançado.
- Inteligência artificial.

Esses recursos poderão ser implementados futuramente.

---

# 25. Observações

O Dashboard Administrativo será o principal ponto de entrada do ERP Comercial Grifres.

Toda a operação da empresa poderá ser acompanhada em uma única tela, fornecendo aos gestores uma visão consolidada do desempenho do negócio, com acesso rápido aos principais módulos e indicadores em tempo real.
