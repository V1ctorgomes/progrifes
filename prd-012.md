# PRD-012 — Cadastro de Clientes

| Campo | Valor |
|--------|--------|
| **Projeto** | ERP Comercial Grifres |
| **Versão** | 1.0.0 |
| **Status** | Aprovado |
| **Prioridade** | Crítica |

---

# 1. Objetivo

Implementar o módulo de Cadastro de Clientes do ERP Comercial Grifres.

Todos os clientes que realizarem um pedido deverão possuir um cadastro próprio no ERP, permitindo centralizar informações, histórico de compras, endereços e relacionamento comercial.

O cadastro deverá ser unificado, evitando duplicidade de clientes.

---

# 2. Objetivos

- Criar cadastro completo de clientes.
- Evitar clientes duplicados.
- Vincular pedidos aos clientes.
- Permitir edição das informações.
- Gerenciar múltiplos endereços.
- Preparar estrutura para CRM.

---

# 3. Escopo

Este PRD contempla:

- Cadastro manual.
- Cadastro automático no checkout.
- Listagem.
- Pesquisa.
- Filtros.
- Edição.
- Exclusão lógica.
- Múltiplos endereços.

Não contempla:

- Histórico de compras.
- CRM.
- Programa de fidelidade.
- Marketing.

---

# 4. Fluxo Geral

```text
Cliente realiza pedido

↓

Sistema verifica telefone

↓

Cliente já existe?

↓

SIM
↓

Atualiza informações

↓

NÃO
↓

Cria novo cadastro

↓

Pedido vinculado ao cliente
```

---

# 5. Dados do Cliente

Campos obrigatórios:

```text
Nome

Telefone
```

Campos opcionais:

```text
E-mail

CPF

Data de nascimento

Observações
```

---

# 6. Endereços

Cada cliente poderá possuir diversos endereços.

Campos:

```text
CEP

Rua

Número

Complemento

Bairro

Cidade

Estado

Referência

Principal
```

Sempre deverá existir apenas um endereço principal.

---

# 7. Identificação

A identificação principal será:

```text
Telefone
```

Caso o telefone já exista:

O sistema deverá atualizar os dados do cliente e reutilizar o cadastro existente.

# 16. Regras de Negócio

- Não permitir telefones duplicados.
- Apenas um endereço principal por cliente.
- Exclusão lógica.
- Cliente nunca será removido caso possua pedidos.
- Pedidos sempre deverão estar vinculados a um cliente.

- CRM.
- Segmentação.
- Fidelidade.
- Campanhas.
- Newsletter.

Esses recursos serão implementados em PRDs futuros.

---

# 22. Observações

Este módulo será a base do relacionamento com os clientes dentro do ERP Comercial Grifres.

Todos os pedidos deverão estar vinculados a um cadastro único de cliente, garantindo consistência dos dados e preparando o sistema para os módulos de CRM, histórico de compras, financeiro e relatórios.
